### Mover o Nginx para o namespace `apps`
Seu deployment atual está rodando no namespace `default`. Vamos recriar corretamente na nova estrutura.

#### Remover deployment antigo do default
```bash
root@rafaelferreiraneves:~# kubectl delete deployment nginx-deployment
deployment.apps "nginx-deployment" deleted from default namespace

root@rafaelferreiraneves:~# kubectl delete service nginx-service
service "nginx-service" deleted from default namespace
```

#### Editar deployment com namespace definido
Abra o arquivo (`vi nginx-deployment.yaml`) e insira `namespace: apps`:
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  namespace: apps
spec:
  replicas: 1
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80
```

#### Editar service com namespace definido
Abra o arquivo (`vi nginx-service.yaml`) e adicione `namespace: apps`:
```yaml
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
  namespace: apps
spec:
  type: NodePort
  selector:
    app: nginx
  ports:
  - port: 80
    targetPort: 80
    nodePort: 30080
```

#### Aplicar novamente os manifestos
```bash
root@rafaelferreiraneves:~# kubectl apply -f nginx-deployment.yaml
deployment.apps/nginx-deployment created

root@rafaelferreiraneves:~# kubectl apply -f nginx-service.yaml
```

## Persistent Volumes (Armazenamento)

### Ver StorageClass
```bash
root@rafaelferreiraneves:~# kubectl get storageclass
NAME                   PROVISIONER             RECLAIMPOLICY   VOLUMEBINDINGMODE      ALLOWVOLUMEEXPANSION   AGE
local-path (default)   rancher.io/local-path   Delete          WaitForFirstConsumer   false                  6h21m
```

### Criar PVC (PersistentVolumeClaim)
```bash
vi nginx-pvc.yaml
```

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: nginx-pvc
  namespace: apps
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 1Gi
```

#### Aplicar PVC
```bash
root@rafaelferreiraneves:~# kubectl apply -f nginx-pvc.yaml
persistentvolumeclaim/nginx-pvc created
```

#### Verificar PVC criado e vinculado
```bash
root@rafaelferreiraneves:~# kubectl get pvc -n apps
NAME        STATUS   VOLUME                                     CAPACITY   ACCESS MODES   STORAGECLASS   VOLUMEATTRIBUTESCLASS   AGE
nginx-pvc   Bound    pvc-0ab336fa-6c1d-481e-84d5-eada22cd27f8   1Gi        RWO            local-path     <unset>                 19m
```

---

### Editar deployment para usar o Volume
```bash
vi nginx-deployment.yaml
```
Substitua pelo conteúdo completo mapeando o volume criado:
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  namespace: apps
spec:
  replicas: 1
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80
        volumeMounts:
        - name: nginx-storage
          mountPath: /usr/share/nginx/html
      volumes:
      - name: nginx-storage
        persistentVolumeClaim:
          claimName: nginx-pvc
```

#### Aplicar deployment atualizado
```bash
root@rafaelferreiraneves:~# kubectl apply -f nginx-deployment.yaml
deployment.apps/nginx-deployment configured
```

#### Verificar pod atualizado
```bash
root@rafaelferreiraneves:~# kubectl get pods -n apps
NAME                               READY   STATUS    RESTARTS   AGE
nginx-deployment-bf54745fd-n8dnq   1/1     Running   0          12m
```

### Criar e Declarar a Aplicação no ArgoCD
Crie o arquivo local definindo onde o ArgoCD deve escutar e buscar atualizações automatizadas:
```bash
vi nginx-app.yaml
```

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: nginx
  namespace: argocd
spec:
  project: default
  source:
    repoURL: [http://172.19.157.82:30090/admin/gitops-apps.git](http://172.19.157.82:30090/admin/gitops-apps.git)
    targetRevision: main
    path: gitops-nginx
  destination:
    server: [https://kubernetes.default.svc](https://kubernetes.default.svc)
    namespace: apps
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
```

#### Aplicar aplicação no cluster
```bash
kubectl apply -f nginx-app.yaml

O Argo CD vai ler o repo Git, entrar em gitops-nginx e ler o deployment.yaml, service.yaml e ingress.yaml, e aplicar no cluster, como se fosse kubectl apply -f automaticamente

```


### 🚀 Ciclo Automatizado Ativo (GitOps)
A partir de agora, qualquer alteração enviada ao repositório Git na pasta `gitops-nginx` será detectada pelo **ArgoCD**, que aplicará as mudanças no cluster automaticamente, mantendo o estado desejado sempre sincronizado.
